RETURN POLICY ‚Äì FRONTEND INTEGRATION GUIDE
==========================================

OVERVIEW
--------
- Each product has its own return window in **days** (`returnPolicyDays`).
- When an order is marked as **Delivered**, the backend stores `deliveredAt` on the order.
- The frontend should:
  - Show the return window for each product.
  - Allow return requests ONLY when the order item is still inside its return window.


PRODUCT RESPONSE (PRODUCT API)
------------------------------
DTO: ProductDto (e.g. from `/api/product`, `/api/product/{id}`, etc.)

Relevant fields:
- `id`: number
- `name`: string
- `price`: number
- `stockQuantity`: number
- `imageUrl`: string | null
- `categoryId`: number
- `categoryName`: string
- `status`: "Pending" | "Approved" | "Rejected"
- `isActive`: boolean
- `createdAt`: string (ISO date)
- `updatedAt`: string | null
- `approvedAt`: string | null
- `approvedByUserId`: number | null
- `approvedByEmail`: string | null
- **`returnPolicyDays`: number (0‚Äì365)**  
  - 0 = NO RETURNS ALLOWED for this product.
  - >0 = number of days after delivery during which user can request a return.

Frontend usage:
- On product detail page, show:
  - If `returnPolicyDays > 0` ‚Üí ‚ÄúReturn within X days of delivery‚Äù.
  - Else ‚Üí ‚ÄúNo returns on this product‚Äù.


ORDER RESPONSE (ORDER API)
--------------------------
DTO: OrderDto (e.g. from `/api/order`, `/api/order/{id}`, `/api/order/admin`)

Relevant fields:
- `id`: number
- `userId`: number
- `userEmail`: string
- `userName`: string
- `status`: "Pending" | "Processing" | "Shipped" | "Delivered" | "Cancelled"
- `totalAmount`: number
- `shippingAddress`: string | null
- `shippingCity`: string | null
- `shippingState`: string | null
- `shippingZipCode`: string | null
- `shippingCountry`: string | null
- `notes`: string | null
- `createdAt`: string (ISO date)
- `updatedAt`: string | null
- **`deliveredAt`: string | null**  
  - Set when admin changes status to **Delivered**.
  - Used to calculate the return window.
- `orderItems`: `OrderItemDto[]`

OrderItemDto:
- `id`: number
- `productId`: number
- `productName`: string
- `price`: number
- `quantity`: number
- `subTotal`: number
- **`returnStatus`**: "None" | "Requested" | "Approved" | "Rejected"
- **`returnRequestedAt`**: string | null (ISO date)
- **`returnResolvedAt`**: string | null (ISO date)
- **`returnReason`**: string | null (user reason and/or resolution note)
- **`refundStatus`**: "None" | "Initiated" | "Done" | "Refunded"
  - Visible to customer on order/order-item. Use to show "return refund status".
  - When return is approved, backend sets this to "Initiated". Admin can update to "Done" or "Refunded".
  - **Refunded** = money has been refunded; user can see "Return refunded" state.


HOW TO CHECK IF AN ITEM IS STILL IN RETURN WINDOW
-------------------------------------------------

For each order item:
1. Ensure the **order** is delivered:
   - `order.status === "Delivered"`
   - `order.deliveredAt` is not null

2. Get the product‚Äôs return policy:
   - From product API (`ProductDto.returnPolicyDays`).
   - You may:
     - Either embed products in your state/store, OR
     - Call product API when rendering order detail.

3. Compute days since delivery:
   - `daysSinceDelivery = (now - new Date(order.deliveredAt)) / (1000 * 60 * 60 * 24)`

4. Determine eligibility:
   - If `product.returnPolicyDays > 0`
   - AND `daysSinceDelivery <= product.returnPolicyDays`
   - THEN **item is within return window**.

5. Frontend behavior:
   - Show **‚ÄúRequest Return‚Äù** button ONLY when item is within return window.
   - Hide/disable the button when:
     - Order is not Delivered, OR
     - `returnPolicyDays === 0`, OR
     - `daysSinceDelivery > returnPolicyDays`.


SUGGESTED UI LOGIC
------------------

On **Order Detail** page:
- For each item:
  - Display:
    - Product name, price, quantity, subtotal.
    - Return text:
      - If return allowed: ‚ÄúReturn within X days of delivery‚Äù.
      - If no returns: ‚ÄúNo returns‚Äù.
  - Show a **‚ÄúRequest Return‚Äù** button when:
    - Order is Delivered.
    - Within return window (as per logic above).

On **My Orders** list:
- Optionally show:
  - A ‚ÄúReturn available until: {deliveredAt + returnPolicyDays}‚Äù label.
  - Or ‚ÄúReturn window expired‚Äù when out of range.


RETURN API (IMPLEMENTED)
------------------------

1) User requests return
   - **POST** `/api/order/return`
   - Auth: Required (Bearer token). Customer requests for their own order.
   - Body:
     {
       "orderItemId": 1,
       "reason": "Size does not fit"
     }
   - Success (200): `{ "success": true, "message": "Return requested successfully. Waiting for seller or admin approval." }`
   - Rules (backend enforces):
     - Order must belong to current user.
     - Order must be Delivered and have `deliveredAt` set.
     - Product must have `returnPolicyDays` > 0.
     - Current date must be within (deliveredAt + returnPolicyDays).
     - Item `returnStatus` must be "None" (not already requested/approved/rejected).

2) Seller/Admin approves or rejects return
   - **POST** `/api/order/return/resolve`
   - Auth: Required. Roles: **Admin** or **Seller**.
   - Body:
     {
       "orderItemId": 1,
       "approved": true,
       "note": "OK to return"
     }
   - Success (200): 
     - If approved: `{ "success": true, "message": "Return approved. Product stock has been restored." }`
     - If rejected: `{ "success": true, "message": "Return rejected." }`
   - Rules:
     - Only Admin or Seller can call.
     - Seller can only resolve returns for **their own products** (product‚Äôs CreatedByUserId = current user).
     - Item `returnStatus` must be "Requested".
     - If approved: product stock is increased; backend sets item `refundStatus` to "Initiated".

3) Admin updates refund status (so customer can see "return refunded")
   - **PUT** `/api/order/return/refund-status`
   - Auth: Required. Role: **Admin** only.
   - Body:
     {
       "orderItemId": 1,
       "refundStatus": "Refunded"
     }
   - Valid `refundStatus` values: **"None"**, **"Initiated"**, **"Done"**, **"Refunded"**.
   - Success (200): `{ "success": true, "message": "Refund status updated successfully" }`
   - Rules:
     - Only allowed when the order item's `returnStatus` is "Approved".
     - Customer sees `refundStatus` on GET order / order by id (in each `orderItems[].refundStatus`).
   - Frontend: Show user-friendly text per status, e.g.:
     - None -> (no refund text or "Pending")
     - Initiated -> "Refund initiated"
     - Done -> "Refund completed"
     - Refunded -> "Return refunded" (recommended label so user clearly sees refund is done)
